Dependacy-Injection ---> 
1.It is design pattern used to increase maintanability and testability of applications by reducing coupling between components.
2.DI works on this fundamental as insted of instantiating objects within a class those objects are passed in as parameters to the class ,like passing it to the constructor or the method ,This allows greater flexibility in the way of objects are created & managed.
3.ASP.net core provides a built in container that can be used to manage the dependancies of an application.The container is responsible for creating and managing instances of services which are registered with container when the appliction starts.
