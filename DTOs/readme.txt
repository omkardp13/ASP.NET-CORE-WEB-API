                                                            DTOs
1.Data Transfer Objects
2.Common Practice To Encapsulate Data & Define What Data Is Send To Or Received From Clients
3.DTO's Helps in decoupling the internal data models from API models enhancing security,performance and maintainability

*Separation of concerns ---> DTO's helps separating the internal data model of your application from data model.
*Security ---> By using DTO's you can control the data exposed to client ,if your model include password or any sensitive info you can exclude these fields from DTO
*Data Sharing--->DTO's allows you shape data that is send or receive from client.
